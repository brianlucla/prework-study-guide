Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The DOCTYPE declaration tells our browser what document type to expect.
  • lang= tells the brower the language for the webpage.
  • The style element defines the CSS styles associated with the webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The charset element sets our character encoding.
  • The http-equiv elemnt sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • When working with visible HTML elements, the content is contained btw an opening tag and a closing tag.
  • Closing tags begin with a forward slash.
  • h1, h2, h3, h4, h5, h6 elements represent the level of heading a given text block represents.
    Use header tags in order from 1-6.
    h1 should only appear once on each page.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items respectively.
  • The img element contains information about images that are displayed on the webpage and can contain various attributes.
  • The src attribute defines the location of the image.
  • The alt attribute contains a text string that describes the appearance and functionality of the image.
  • The br element creates a line of empty space.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages.
  • The link element creates a connection to an external file (e.g. use to link to CSS file)
  • The href attribute points to the URL for the link.
  • The footer element contains text at the bottom of the webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A period before a selector defines the selector as a class selector. Any HTML element assigned to the class will gain these property assignments from the CSS.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot.
  • git pull: fetch and merge any commits from the tracking remote branch
  • git pull origin main: origin keyword indicates the source of the pull will be in the GitHub repo. After origin, you must specify the branch to pull from
  • git add: adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add ALL changes
